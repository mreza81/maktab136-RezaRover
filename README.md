## 📖 درباره پروژه

**Car-Commerce** یک راه‌حل کامل و یکپارچه برای نمایش، جستجو و خرید‌و‌فروش خودرو به‌صورت آنلاین است. این پلتفرم با بهره‌گیری از تکنولوژی‌های مدرن وب — **Next.js**، **React**، **TypeScript** و **TailwindCSS** — تجربه‌ای روان، سریع و امن را برای کاربران فراهم می‌کند.

از ویژگی‌های کلیدی پروژه می‌توان به مدیریت آگهی‌ها، جستجوی پیشرفته، بارگذاری تصاویر توسط فروشندگان و یک **پنل ادمین قدرتمند** اشاره کرد که امکان مدیریت کامل سیستم را به مدیران می‌دهد.

## 🌟 پنل ادمین

پنل ادمین به‌عنوان قلب مدیریتی پلتفرم طراحی شده و امکانات زیر را در اختیار مدیران قرار می‌دهد:

- **📊 داشبورد مدیریتی** — نمایش آمار کلی سایت (تعداد کاربران، خودروها، فروش‌ها)
- **🚗 مدیریت خودروها** — مشاهده، ویرایش، تایید/رد و حذف آگهی‌ها
- **🏷️ مدیریت دسته‌بندی‌ها و برندها** — افزودن، ویرایش و حذف دسته‌بندی‌های خودرو
- **📦 مدیریت سفارشات** — مشاهده و تغییر وضعیت سفارش‌ها
- **🔐 دسترسی محدود** — ورود به پنل ادمین فقط برای کاربران با نقش Admin

---

## 🔐 امنیت و کنترل دسترسی

امنیت یکی از ارکان اصلی طراحی این پروژه بوده و در چند لایه مستقل پیاده‌سازی شده است:

- **میدل‌ویر (Middleware):** پیش از رسیدن درخواست به هر صفحه یا روت محافظت‌شده، میدل‌ویر Next.js نقش، توکن و وضعیت احراز هویت کاربر را بررسی می‌کند. در صورت نبود دسترسی کافی (مثلاً تلاش یک کاربر عادی برای ورود به مسیرهای `/admin`)، درخواست قبل از رندر شدن صفحه ریدایرکت یا رد می‌شود.
- **اینترسپتور (Interceptor):** تمامی درخواست‌های ارسالی به سمت سرور از یک لایه اینترسپتور عبور می‌کنند که وظیفه افزودن خودکار توکن احراز هویت به هدر درخواست‌ها، مدیریت یکپارچه خطاها (مانند انقضای توکن یا عدم دسترسی) و در صورت نیاز، تلاش مجدد یا خروج خودکار کاربر را بر عهده دارد.
- **محافظت سمت سرور:** علاوه بر بررسی‌های سمت کلاینت، مسیرهای حساس API نیز در سمت سرور نقش و هویت کاربر را مجدداً اعتبارسنجی می‌کنند تا دور زدن لایه‌های امنیتی کلاینتی ممکن نباشد.

ترکیب این لایه‌ها باعث می‌شود حتی در صورت دستکاری مستقیم درخواست‌ها از سمت کاربر، دسترسی غیرمجاز به منابع حساس (مانند پنل ادمین یا API‌های مدیریتی) امکان‌پذیر نباشد.

## ⚡ استراتژی‌های رندرینگ در Next.js

این پروژه از رویکردهای مختلف رندرینگ Next.js برای بهینه‌سازی عملکرد و تجربه کاربری بهره می‌برد:

| استراتژی | کاربرد | مثال |
|---|---|---|
| **SSR** (Server-Side Rendering) | صفحاتی با داده‌های پویا و اختصاصی به هر درخواست | `/cars/[id]` |
| **CSR** (Client-Side Rendering) | بخش‌های تعاملی که نیازی به داده فوری ندارند | فرم بارگذاری عکس، داشبورد کاربر |
| **SSG** (Static Site Generation) | صفحاتی با محتوای ثابت و کم‌تغییر | صفحه اصلی، صفحه درباره ما |
| **ISR** (Incremental Static Regeneration) | ترکیب SSG با به‌روزرسانی دوره‌ای | لیست خودروهای پرفروش |

انتخاب استراتژی مناسب برای هر صفحه بر اساس نیاز آن صفحه به داده، سئو و عملکرد صورت گرفته است.

## ⚙️ نصب و راه‌اندازی

برای اجرای پروژه به‌صورت لوکال، مراحل زیر را دنبال کنید:

\`\`\`bash
# کلون کردن مخزن
git clone <REPO_URL>

# ورود به پوشه پروژه
cd car-commerce

# نصب وابستگی‌ها
npm install

# اجرای پروژه در محیط توسعه
npm run dev
\`\`\`

پروژه پس از اجرا روی آدرس `http://localhost:3000` در دسترس خواهد بود.
